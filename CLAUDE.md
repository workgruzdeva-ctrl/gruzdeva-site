# Сайт Валентины Груздевой — заметки для будущих агентов

Это статический сайт на GitHub Pages: `https://workgruzdeva-ctrl.github.io/gruzdeva-site/`.
Деплой через `git push` в `main`. CDN/Cloudflare нет.

## Workflow

После любых правок:

```bash
git add <файлы>
git commit -m "коротко по-русски, что изменилось"
git push
```

GH Pages обновится за 1-3 минуты. Если пользователь не видит изменений — Cmd+Shift+R или инкогнито (это всегда кеш браузера).

**Cache-busting:** для общих JS/CSS файлов (`mascot.js`, `challenge.js`, `share-utils.js`, `day.css`) используется query string `?v=N`. При значимых правках бампайте версию во всех местах подключения.

## Структура страниц

**Корневые лендинги:**
- `index.html` — главный сайт (Hero, консультации, материалы, тарифы)
- `group.html` — групповой тренинг СДВГ (6 недель, 240 €)
- `course.html` — лендинг «Курс молодого птенца» (€25)
- `club.html` — клуб «Стая» по подписке
- `test.html` — самотест СДВГ
- `prep.html` — подготовка к первой встрече

**Курс молодого птенца:**
- `course-home.html` — личный кабинет курса (5 уроков, прогресс в `localStorage.ptenec_progress`)
- `lesson-1.html` … `lesson-5.html` — уроки (видео-плейсхолдер + действие + кнопка «Пройдено»)
- `course-complete.html` — выпускной экран с конфетти и фанфарами

**Закрытая зона участниц группы:**
- `m/grp/` — личные кабинеты для каждой участницы. URL вида `?u=<slug-XXXX>&n=<имя>`
- `m/grp/mascot.js` — 7 стадий гусёнка (Яйцо → Птенец → … → Боевой гусь). Подключается с `?v=N`.
- `m/grp/birds/` — 9 PNG-птичек (Королева стаи + остальные стадии), 1200×1200, прозрачный фон, центрированы по X, выровнены по нижнему краю
- Стадия маскота = `state.taskAwardKeys.length`. Каждая отмеченная weekly-task = +1 стадия (накопительно, не уменьшается при снятии галочки)

**Личный кабинет одного клиента:**
- `m/personal/` — шаблон. URL `?n=<имя>` подставляет имя в приветствие. Прогресс в `localStorage.personal_v1`
- 8 задач (2 больших + 6 маленьких), у каждой шаги. Птенец растёт за каждый шаг
- Кнопка «PDF / печать» — `window.print()` с `@media print`-стилями
- Сейчас задачи захардкожены под одного клиента (ПМЖ через помогатора, спорт, бытовые дела). Если придёт второй клиент — копировать в `m/personal-2/`

**Челлендж «Стаей идти веселее»:**
- `challenge/` — 9-дневный мини-челлендж, воронка перед групповым тренингом
- Дни: PVT / Go-No-Go / стол / N-back / сторис / Висконсин / опрос близких / Flanker / бинго
- Прогресс в `localStorage.gruzdeva_challenge_v1`. Между днями замок до полуночи следующего календарного дня
- `?dev=1` снимает замок и автоматически регистрирует «Тест»-профиль
- `challenge/share-utils.js` — единая отрисовка PNG-картинок для сторис (1080×1920) с птичкой-королевой

## Tribute / оплата

Все CTA-кнопки «Купить» в `course.html`, `club.html` используют атрибут `data-buy`. Внизу страницы один скрипт:
```js
const PAY_LINK = "https://t.me/dr_gruzdeva";  // ← подставить Tribute-ссылку
document.querySelectorAll('[data-buy]').forEach(a => a.href = PAY_LINK);
```
Чтобы поменять реквизиты — одна строка.

## Работа с артефактами из Claude.ai

Когда пользователь приносит HTML-файл из артефактов Claude (Omelette):
1. Файл обычно 5+ МБ — внутри React + Babel + base64-фонты + панель Tweaks
2. Делегируйте парсинг подагенту, он сам почистит и положит в нужное место
3. Картинки птиц подменяйте на `m/grp/birds/<N>_*.png` если в оригинале placeholder
